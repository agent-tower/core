# Agent Tower - 交互设计文档

## 1. 整体布局

采用经典的双栏布局：左侧任务列表 + 右侧详情面板。

```
┌─────────────────────────────────────────────────────────────────────┐
│  Agent Tower                         [切换项目 ▼] [全部项目] [+ 任务]│
├────────────────────────────┬────────────────────────────────────────┤
│                            │                                        │
│      任务列表               │              详情面板                   │
│      (左侧面板)             │             (右侧面板)                  │
│                            │                                        │
└────────────────────────────┴────────────────────────────────────────┘
```

---

## 2. 左侧面板 - 任务列表

### 2.1 状态分组（手风琴）

任务按状态分组，使用手风琴交互，可展开/折叠：

| 状态 | 默认 | 说明 |
|------|------|------|
| 待审查 | **展开** | 最重要，需要用户关注 |
| 执行中 | 展开 | 正在进行的任务 |
| 待执行 | 折叠 | 队列中等待执行（为并发限制预留） |
| 已完成 | 折叠 | 历史记录 |

### 2.2 任务状态图标

不使用进度条，使用简洁图标表示状态：

```
◐  待审查 (半圆)
◉  执行中 + ⟳ (旋转动画)
○  待执行 (空心圆)
✓  已完成 (勾选)
```

### 2.3 多项目混合显示

支持多个项目的任务混合显示，通过以下方式区分：

1. **项目前缀**: `项目名/任务名` 格式
2. **颜色标识**: 每个项目分配独立颜色
3. **筛选功能**: 顶部提供项目切换下拉菜单和"全部项目"按钮

### 2.4 完整示例

```
┌────────────────────────────┐
│                            │
│  ▼ 待审查 (2)              │
│    ◐ Tower/API重构         │
│    ◐ Web/表单验证          │
│                            │
│  ▼ 执行中 (3)              │
│    ◉ Tower/登录功能   ⟳   │
│    ◉ SDK/类型修复     ⟳   │
│    ◉ Web/首页优化     ⟳   │
│                            │
│  ▶ 待执行 (5)              │
│                            │
│  ▶ 已完成 (12)             │
│                            │
└────────────────────────────┘
```

## 3. 右侧面板 - 详情区域

### 3.1 头部信息

显示当前选中任务的基本信息：

```
┌────────────────────────────────────────────────────────────────┐
│  Tower / API重构                                               │
│  Agent: Claude Code    分支: feat/api    ⟳ 执行中             │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  ┌─ 任务描述 ──────────────────────────────────────────────┐  │
│  │ 重构 API 路由，将用户相关接口拆分到独立模块             │  │
│  └─────────────────────────────────────────────────────────┘  │
```

### 3.2 AI 交互内容

这是详情面板的核心区域，展示 Agent 的执行过程。

#### 内容类型与图标

| 图标 | 类型 | 说明 |
|------|------|------|
| ◆ | Agent 动作 | 正在做什么 |
| ◇ | Agent 说明 | 解释/思考结果 |
| ▶ | 工具调用 | 可点击展开详情 |
| █ | 光标 | 正在输出 |

#### 折叠规则

- **Thinking 内容**: 默认折叠
- **工具调用**: 默认折叠，点击展开查看详情
- **Agent 说明文字**: 默认展开

#### 用户消息显示

用户发送的追加指令以区分样式显示：

```
│  ◇ 已完成路由拆分...                                          │
│                                                                │
│  ┌─ 你 ────────────────────────────────────────────────────┐  │
│  │ 请同时添加 JSDoc 注释                                    │  │
│  └─────────────────────────────────────────────────────────┘  │
│                                                                │
│  ◆ 收到，正在为每个路由函数添加 JSDoc...                      │
```

### 3.4 底部操作栏

| 按钮 | 功能 | 显示条件 |
|------|------|----------|
| 发送 | 向 Agent 发送追加指令 | 始终显示 |
| 停止 | 终止当前任务执行 | 执行中时显示 |

## 4. 工作流

当前版本采用简化工作流：

```
创建任务 → 立即执行 → 待审查 → 完成
```

"待执行"状态为后续并发限制功能预留，当前默认折叠。

---

## 5. 设计决策记录

| 决策 | 理由 |
|------|------|
| 不使用进度条 | AI Agent 进度难以量化，假进度条误导用户 |
| 待审查默认展开 | 这是用户最需要关注的状态 |
| 工具调用默认折叠 | 减少视觉噪音，按需查看详情 |
| 多项目混合显示 | 避免频繁切换项目，提高效率 |
| 项目前缀+颜色 | 在混合视图中快速区分任务归属 |
